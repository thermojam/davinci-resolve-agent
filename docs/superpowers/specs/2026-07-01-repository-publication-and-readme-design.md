# Публикация репозитория и защита main — дизайн

## Цель

Сделать `thermojam/davinci-resolve-agent` публичным, объединить навык `davinci-console-bridge` со всеми его правилами, скриптами, шаблонами и справочниками в `main`, добавить понятный русскоязычный README и защитить `main` от прямых изменений, force-push и удаления.

## Выбранный подход

Работа продолжается в существующей ветке `codex/davinci-console-bridge-skill` и PR №1. README добавляется в эту же ветку, после проверки PR объединяется в `main`. Защита включается последней, чтобы не заблокировать первоначальное объединение.

Политика `main`:

- изменения только через Pull Request;
- force-push запрещён;
- удаление ветки запрещено;
- обязательные approvals и status checks пока не включаются, потому что репозиторий индивидуальный и CI ещё отсутствует;
- правила распространяются и на администратора репозитория.

## README

README на русском объясняет новичку:

- назначение навыка и его ограничения;
- требования к DaVinci Resolve, Codex и исходным данным;
- установку навыка в `~/.codex/skills`;
- вызов `$davinci-console-bridge`;
- обязательный опрос, план и подтверждение перед изменением Resolve;
- работу через встроенную `Workspace > Console > Py3`;
- проверку ориентации и параметров экспорта;
- типовые ошибки и безопасное восстановление;
- структуру репозитория и запуск тестов.

## Проверка

- автоматический тест проверяет наличие README и ключевых инструкций для новичка;
- полный набор тестов навыка проходит;
- GitHub показывает репозиторий как `PUBLIC`;
- PR №1 имеет состояние `MERGED`;
- `origin/main` содержит README и каталог `skills/davinci-console-bridge`;
- API GitHub подтверждает запрет force-push/удаления и обязательный PR.

## Безопасность

Перед переключением видимости выполняется проверка отслеживаемых файлов на токены, приватные ключи и файлы окружения. Пользовательские исходники DaVinci и локальные credential-файлы не добавляются в Git.
