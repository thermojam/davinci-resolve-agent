# DaVinci Console Bridge для Codex

`davinci-console-bridge` — навык Codex для подготовки, монтажа, диагностики и экспорта проектов DaVinci Resolve. Он рассчитан прежде всего на DaVinci Resolve Free в macOS и использует встроенную Python-консоль Resolve.

Навык не начинает монтаж сразу. Сначала он собирает исходные данные, проверяет файлы и технические ограничения, показывает план и ждёт вашего подтверждения.

## Что умеет навык

- проверять наличие видео, аудио, изображений, шрифтов, субтитров и логотипов;
- уточнять сценарий, длительность, стиль, референсы и права на музыку;
- составлять план монтажа до изменения проекта;
- создавать Python-задачи для встроенной консоли DaVinci Resolve;
- проверять горизонтальную, вертикальную и квадратную ориентацию;
- готовить отдельные варианты таймлайна под разные площадки;
- проверять разрешение, FPS, кодеки, звук, длительность и размер экспорта;
- диагностировать типовые ошибки связи Codex и Resolve.

## Что потребуется

1. macOS с запущенным DaVinci Resolve.
2. Codex с доступом к локальным файлам проекта.
3. Абсолютный путь к исходникам.
4. Сценарий или описание желаемого результата.
5. Референсы либо объяснение, какие их свойства нужно повторить.
6. Название площадки и конкретного размещения: например, YouTube Shorts, Instagram Reels или обычное горизонтальное видео.
7. Право использовать предоставленные материалы.

`ffmpeg` и `ffprobe` нужны для технической проверки готовых файлов. На macOS их обычно устанавливают командой `brew install ffmpeg`.

## Установка

Склонируйте репозиторий и скопируйте каталог навыка в личный каталог Codex:

```bash
git clone https://github.com/thermojam/davinci-resolve-agent.git
cd davinci-resolve-agent
mkdir -p "$HOME/.codex/skills/davinci-console-bridge"
rsync -a --delete skills/davinci-console-bridge/ "$HOME/.codex/skills/davinci-console-bridge/"
```

Перезапустите Codex или откройте новый диалог, чтобы навык появился в списке доступных.

## Как запустить

В любом проекте Codex напишите:

```text
$davinci-console-bridge
```

Можно сразу добавить задачу:

```text
$davinci-console-bridge
Исходники находятся в /абсолютный/путь/к/video.
Нужно сделать заставку длительностью 20–30 секунд для YouTube.
```

Не вставляйте пароли, токены и другие секреты в запрос или в папку с исходниками.

## Какое поведение ожидать

Навык проходит обязательные этапы:

1. **Опрос.** Codex задаёт по одному короткому вопросу об исходниках, сценарии, референсах, звуке, формате и площадке.
2. **Проверка.** Файлы и окружение исследуются без изменения проекта Resolve.
3. **Технический профиль.** Для конкретной площадки проверяются актуальные официальные требования и дата проверки.
4. **Геометрия.** Согласуются ориентация, размеры кадра, безопасные зоны и отдельные версии таймлайна.
5. **План.** Codex показывает структуру монтажа, выходные файлы и критерии готовности.
6. **Подтверждение.** Resolve не изменяется до явного подтверждения плана пользователем.
7. **Выполнение.** Codex готовит проверяемый Python-скрипт и команду для консоли Resolve.
8. **Проверка результата.** Экспорт анализируется через `ffprobe` и сравнивается с согласованным профилем.

Если данных недостаточно, навык остановится и задаст уточняющий вопрос. Он не должен молча придумывать творческие решения, заменять существующий проект или считать параметры соцсети неизменными навсегда.

## Связь с DaVinci Resolve

Проверенный способ для Resolve Free:

1. Запустите Resolve и откройте рабочий проект.
2. Выберите `Workspace > Console > Py3`.
3. Вставьте команду, которую подготовил Codex.
4. Нажмите Enter и пришлите Codex полный результат выполнения.

Команда передаёт готовый объект `resolve` во внешний скрипт:

```python
p = r"/absolute/path/to/task.py"
g = {"__file__": p, "__name__": "resolve_console_task", "resolve": resolve}
exec(open(p, encoding="utf-8").read(), g)
g["main"]()
```

Внешнее подключение через `DaVinciResolveScript.scriptapp("Resolve")` не считается рабочим по умолчанию: доступность зависит от версии Resolve и настроек scripting.

## Ориентация и экспорт

До рендера навык должен проверить:

- ориентацию: `landscape`, `portrait` или `square`;
- размеры таймлайна и экспортируемого файла;
- частоту кадров и длительность;
- видео- и аудиокодеки;
- частоту дискретизации и число аудиоканалов;
- ограничение размера файла для выбранной площадки.

Горизонтальная, вертикальная и квадратная версии создаются отдельными таймлайнами или рендерами. Кадрирование выполняется осознанно; изображение нельзя просто растянуть или повернуть.

## Типовые ошибки

### `No such file or directory: 'ffprobe'`

Встроенная консоль Resolve может не видеть shell `PATH`. Установите `ffmpeg` и используйте проверенный абсолютный путь, обычно `/opt/homebrew/bin/ffprobe`.

### `No module named 'DaVinciResolveScript'`

Не импортируйте внешний модуль внутри Py3 Console. Используйте уже доступный глобальный объект `resolve` и команду из раздела выше.

### `DaVinci Resolve is not running or external scripting is disabled`

Для Resolve Free используйте встроенную `Workspace > Console > Py3`. Сам факт успешного импорта модуля не доказывает, что внешнее подключение работает.

### Консоль недоступна в Project Manager

Сначала откройте любой проект, затем откройте консоль через меню `Workspace`.

### `could not set project setting ...`

Не повторяйте неизвестный ключ вслепую. Скрипт должен прочитать `GetSetting()`, свериться с установленной документацией API и остановиться при неоднозначном состоянии.

### Проект с таким именем уже существует

Навык не удаляет и не заменяет проекты автоматически. Нужно явно выбрать другое имя либо разрешить работу с существующим проектом после проверки его состояния.

## Безопасность

- Существующие проекты считаются неизменяемыми без явного разрешения.
- До подтверждения плана допускается только чтение и диагностика.
- При неясном результате повторного запуска скрипт должен остановиться, чтобы не создать дубликаты таймлайнов или медиа.
- Исходники, рендеры и credential-файлы не должны попадать в Git.
